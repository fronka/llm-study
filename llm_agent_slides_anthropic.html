よくある誤解

LLMが「考えて」回答を組み立てている

正しい理解

毎回独立したトークン予測の連続。内部状態・意図・記憶はない。

02・03 — 外部接続

ツール呼び出しとRAG

RAGは「学習」ではない。LLMはテキストを出力するだけ——アプリがそれをパースして実行する。

SYSTEM DIAGRAM — RAG / ツール呼び出し

よくある誤解

「RAGはAIに社内データを学習させる技術」

正しい理解

モデルの重みは変わらない。LLMはJSON形式のテキストを出力し、アプリがパースしてDBを検索する。

3 / 5
04 — エージェントの本質

ループ：Agentがやっていること

LLMのテキスト出力→アプリが実行→結果をContextに追加→再びLLMへ。このループをアプリが制御する。

SYSTEM DIAGRAM — エージェントループの実態

誤解されがちな点

「AIがループを制御している」

実際の構造

ループ・上限・タイムアウトはすべて外部のアプリ（オーケストレーター）が管理。LLMはその都度テキストを返すだけ。

4 / 5
05 — マルチエージェント

マルチエージェント：役割分担

エージェント同士の連携も、すべてテキスト（メッセージ）のやり取り。アプリが全体を制御する。

SYSTEM DIAGRAM — マルチエージェントの実態

コンテキスト共有の有無で分類
